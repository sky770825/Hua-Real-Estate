# 常見問題 FAQ

## 🚨 緊急問題

### Q: 伺服器完全無法啟動怎麼辦？
**A**: 執行快速修復腳本
```bash
./快速修復腳本.sh
```

或手動執行：
```bash
lsof -ti:3001 | xargs kill -9
rm -rf .next
npm run dev
```

---

### Q: 頁面顯示 "Cannot find module" 錯誤？
**A**: 這是 Next.js 構建快取問題
```bash
rm -rf .next
npm run dev
```

---

### Q: F5 刷新後頁面無法顯示？
**A**: 
1. 清除瀏覽器快取 (`Ctrl+Shift+Delete`)
2. 清除構建快取：`rm -rf .next`
3. 重啟伺服器：`npm run dev`

---

## 📅 日期相關

### Q: 日期選擇器顯示去年的日期？
**A**: 已修復！現在只顯示未來日期（從今天或最近的週四開始）

### Q: 日期選項太多怎麼辦？
**A**: 系統預設只顯示未來 3 個月，點擊「更多」按鈕可載入更多日期

### Q: 如何選擇更遠的日期？
**A**: 點擊日期選擇器旁邊的「更多」按鈕，每次可載入額外 3 個月的日期

---

## 🗑️ 刪除功能

### Q: 會議刪除後仍然顯示？
**A**: 
1. 檢查瀏覽器控制台是否有錯誤 (F12)
2. 清除快取（系統設定 → 清除快取）
3. 重新載入頁面

### Q: 刪除會議時出現錯誤？
**A**: 
- 查看錯誤訊息（會顯示在頁面上方）
- 確認會議是否存在
- 檢查資料庫連接

---

## 💾 資料問題

### Q: 資料顯示過時？
**A**: 
1. 清除快取（系統設定 → 清除快取，密碼：888）
2. 強制重新載入頁面 (`Ctrl+Shift+R`)
3. 重啟伺服器

### Q: 會員列表無法載入？
**A**: 
```bash
# 測試 API
curl http://localhost:3001/api/members

# 如果失敗，檢查：
# 1. 伺服器是否運行
# 2. 資料庫連接是否正常
# 3. 環境變數是否正確
```

---

## 🔐 開發者功能

### Q: 如何解鎖開發者功能？
**A**: 
1. 前往「系統設定」頁面
2. 點擊「解鎖開發者功能」
3. 輸入密碼：`888`

### Q: 忘記開發者密碼？
**A**: 密碼是 `888`（固定密碼）

---

## 🌐 網路問題

### Q: API 請求失敗？
**A**: 
1. 檢查伺服器是否運行：`lsof -ti:3001`
2. 測試 API：`curl http://localhost:3001/api/members`
3. 查看瀏覽器 Network 標籤的錯誤詳情

### Q: 請求過於頻繁錯誤 (429)？
**A**: 
- 等待 1-2 分鐘後再試
- 檢查是否有無限循環的請求
- 清除快取後重試

---

## 🛠️ 技術問題

### Q: TypeScript 編譯錯誤？
**A**: 
```bash
npm run lint
# 查看詳細錯誤訊息並修復
```

### Q: 依賴安裝失敗？
**A**: 
```bash
rm -rf node_modules
npm install
```

### Q: 記憶體不足錯誤？
**A**: 
```bash
NODE_OPTIONS="--max-old-space-size=4096" npm run dev
```

## 📊 效能問題

### Q: 頁面載入很慢？
**A**: 
1. 清除瀏覽器快取
2. 清除伺服器快取（系統設定）
3. 檢查網路連接
4. 檢查是否有大量資料需要載入

### Q: API 回應很慢？
**A**: 
- 檢查資料庫查詢效能
- 確認快取機制正常運作
- 檢查是否有過多的並行請求

---

## 🔍 除錯技巧

### Q: 如何查看詳細錯誤？
**A**: 
1. **瀏覽器控制台** (F12 → Console)
2. **網路請求** (F12 → Network)
3. **伺服器終端機** (查看 npm run dev 的輸出)

### Q: 如何測試 API？
**A**: 
```bash
# 使用 curl
curl http://localhost:3001/api/members

# 使用瀏覽器
# 直接訪問 http://localhost:3001/api/members
```

### Q: 如何檢查資料庫連接？
**A**: 
1. 檢查 `.env.local` 檔案
2. 確認 Supabase Dashboard 狀態
3. 測試 API 端點是否返回資料

## 📝 最佳實踐

### 日常使用建議
1. **定期清除快取**：每週清除一次瀏覽器快取
2. **檢查伺服器狀態**：使用前確認伺服器正常運行
3. **備份資料**：重要操作前先備份（系統設定 → 匯出數據）

### 遇到問題時的標準流程
1. ✅ 檢查伺服器是否運行
2. ✅ 清除快取（瀏覽器 + 伺服器）
3. ✅ 重啟伺服器
4. ✅ 查看錯誤日誌
5. ✅ 參考問題排除指南

---

## 🆘 仍無法解決？

如果以上方法都無法解決問題：

1. **收集資訊**：
   - 錯誤訊息截圖
   - 瀏覽器控制台錯誤
   - 伺服器終端機錯誤
   - 操作步驟

2. **檢查日誌**：
   - 瀏覽器控制台 (F12)
   - 伺服器終端機
   - Supabase Dashboard 日誌

3. **重現問題**：
   - 記錄重現步驟
   - 確認問題是否持續發生
   - 檢查是否與特定操作相關
